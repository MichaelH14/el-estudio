# Base de conocimiento Blender — Índice

Fase de LA HERRAMIENTA: cómo operar **Blender 5.2 LTS** para producir game assets (interfaz, edición, modificadores, nodos, sculpt, export a Unity, addons, la API `bpy` y operación por MCP). La teoría de QUÉ modelar vive aparte en `modelado/` — estos archivos la asumen y no la repiten. Cargar SOLO lo relevante: `blender-mcp-operativo` + `interfaz-flujo` casi siempre al empezar una sesión con un Blender vivo; luego el archivo de la tarea concreta. Cada archivo abre con su trigger ("Cuando cargar este archivo") y cierra con Reglas prácticas + Errores comunes + Fuentes. `[ver: slug]` cruza esta carpeta; `[ver: modelado/slug]`, `[ver: pipeline/slug]` y `[ver: gamedev/slug]` a las bases hermanas.

| Archivo | Qué contiene | Cuándo cargarlo |
|---|---|---|
| `interfaz-flujo.md` | Modelo mental (escena→colecciones→objetos→object data), object vs edit mode, workspaces/editores, los ~30 atajos que valen el 90%, selección (loops/rings), transformaciones y pivots, snapping y 3D cursor, preferencias y unidades métricas para game dev | Antes de operar Blender por primera vez en una sesión, o al navegar UI / seleccionar / transformar / configurar escena |
| `edicion-malla.md` | Herramientas core de Edit Mode (extrude, inset, bevel, loop cut, knife, merge/dissolve, bridge, fill), proportional editing, normales (recalcular/sharp/smooth by angle en 5.x), limpieza de malla, selección avanzada, retopo nativa (shrinkwrap/poly build) | Al modelar, retopologizar, limpiar o reparar la malla de un asset con herramientas nativas |
| `modificadores.md` | Filosofía no-destructiva y orden del stack, los canónicos uno a uno (Mirror, Subdivision, Bevel+weighted normals, Boolean, Array, Solidify, Shrinkwrap, Decimate, Remesh, Triangulate, Data Transfer, Weld), recetas de stacks típicos, aplicar en export, crease/bevel weight | Al construir/ordenar/aplicar/depurar modificadores, o al decidir modificador clásico vs Geometry Nodes |
| `geometry-nodes.md` | Modelo mental (fields/atributos/instancias), casos de juego concretos (scatter, variaciones procedurales, kits paramétricos, geometría por curva), instancias vs realized (crítico para export FBX), node groups como assets, cuándo NO usarlo | Al hacer scatter/variaciones/kits/geometría por curva — y SIEMPRE antes de exportar a Unity un objeto con modificador Geometry Nodes |
| `sculpt.md` | Cuándo esculpir vs poly-modeling, brushes clave por fase (draw/clay strips/crease/dam standard/grab/smooth), Dyntopo vs Multires vs Voxel Remesh, flujo de bloqueo primario→secundario→terciario, máscaras/face sets, simetría, de sculpt a game asset (decimate vs retopo) | Antes de entrar a Sculpt Mode (orgánico, criaturas, rocas, detalle high-poly para bake) o de decidir el método de densidad |
| `import-export.md` | FBX vs glTF a Unity, los settings EXACTOS de export FBX (escala 100x, ejes Y-up/-Z-forward, apply transforms), el bug de rotación -89.98 y cómo evitarlo, qué exporta y qué no (modificadores, instancias GN, materiales), naming/sockets/colliders por convención, export batch, checklist de validación post-export | Al sacar cualquier asset de Blender a Unity, montar un export batch, o diagnosticar escala/rotación/normales/materiales rotos en Unity |
| `addons-ecosistema.md` | El sistema de extensiones 4.2+ (extensions.blender.org, manifiesto, compatibilidad), addons rotos por release (el dolor real), los integrados que activar (Node Wrangler), los famosos por categoría con precio y veredicto para game assets, gratis vs pago (la respuesta honesta), criterios de compra, riesgos (malware/abandono), config y migración | Antes de instalar/activar/evaluar un addon, cuando uno falla tras actualizar Blender, o al decidir si comprar una herramienta de pago |
| `bpy-scripting.md` | Por qué Blender es 100% automatizable, bpy.data vs bpy.ops vs bpy.context, automatizar lo repetitivo (renombrar/export batch/validar), text editor y consola, escribir un addon mínimo, headless (`--background --python`), modificar Blender de verdad (scripts vs fork C++), leer la API grabando con el info editor | Al escribir/ejecutar Python en Blender (MCP, consola, headless), crear/instalar un addon, o elegir entre bpy.data y bpy.ops |
| `blender-mcp-operativo.md` | El setup de este stack (addon + servidor MCP `blender`), arquitectura del socket (puerto 9876, Blender abierto), herramientas que expone el MCP, el flujo del agente (cambio→verificar→seguir), seguridad de execute-code, alternativa headless batch, límites, verificación visual sin ojos (render + medidas por código) | Al operar un Blender vivo por MCP (crear/editar/consultar/screenshot) o al decidir MCP interactivo vs headless batch — cargar SIEMPRE primero al usar el MCP |
| `materiales-preview.md` | Cómo EVALUAR el modelo visualmente en Blender (NO es texturizado): viewport shading (matcaps/cavity/wireframe), material preview vs rendered, materiales planos de trabajo (los slots que Unity verá), EEVEE vs Cycles para game dev, chequeos de silueta/AO/topología/tris, luces neutras, la trampa del render bonito | Al evaluar visualmente un modelo durante el modelado o montar materiales planos de trabajo (slots para Unity) |
| `organizacion-blend.md` | Colecciones (estructura final/high/low/refs/WIP), naming consistente, object data compartida e instancias (alt+D), orphan data y purge, un .blend por asset vs biblioteca linked, Asset Browser, versionado (LFS, _v001, .blend1/autosave), recuperación de desastres | Al crear/estructurar un .blend, limpiar archivos que engordan, decidir .blend por asset vs biblioteca, o versionar/recuperar |
| `blender5-actualidad.md` | Qué cambió de 4.x a 5.x para game assets, cadencia y LTS (cuándo actualizar), la Blender Foundation y el modelo open source (financiación, Development Fund), cómo se desarrolla/contribuye, el ecosistema (Blender Studio, Superhive, comunidades), Blender vs alternativas para game art, recursos de aprendizaje por nivel | Al decidir versión o si actualizar, al toparte con tutoriales que no cuadran con la UI actual, al reportar un bug, o al evaluar alternativas/recursos |
