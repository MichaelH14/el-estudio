# Base de conocimiento Pipeline — Índice

Esta base es el puente aplicado entre las dos hermanas: `../gamedev/` cubre el QUÉ/POR QUÉ de diseño y `../unity/` el CÓMO del motor; aquí vive la UNIÓN — recetas completas, decisiones que se congelan el día 1 y el proceso del dúo humano+agente que lleva un juego de idea a publicado. Cada archivo arranca con "Cuando cargar este archivo": cargar SOLO los relevantes a la tarea, no toda la base. Las referencias `[ver: slug]` apuntan a esta carpeta; `[ver: gamedev/slug]` y `[ver: unity/slug]` a las bases hermanas — estos archivos NO repiten ese contenido, lo asumen. Puntos de entrada: `pipeline-completo` para saber qué toca ahora, `produccion-solo-dev` para cómo trabaja el dúo en cada sesión. Verificado contra Unity 6.x + URP (estado a julio 2026).
