# Base VFX y Shaders — Índice

Cada archivo es autocontenido y abre con una línea «Cuando cargar este archivo»: cargar SOLO los relevantes a la tarea, no toda la base. Las referencias `[ver: slug]` apuntan a otro archivo de esta carpeta; `[ver: unity/…]`, `[ver: pipeline/…]`, `[ver: gamedev/…]`, `[ver: arte-2d/…]` apuntan a las bases hermanas. Aquí va el CÓMO del efecto (qué módulo, qué nodo, qué parámetro); el PORQUÉ del feel vive en `gamedev/game-feel`, la implementación base del feel (shake, hit-stop, tween, pooling) en `pipeline/feel-en-unity`, y el pipeline gráfico + rendimiento en `unity/rendering-urp` y `unity/rendimiento-unity`.
Reparto interno: `shader-graph-fundamentos` = la mecánica del grafo; `recetas-shaders` = shaders concretos; `particulas-vfx-graph` = partículas Shuriken/GPU; `vfx-recetas` = efectos de juego multicapa; `vfx-2d` = el ángulo 2D/pixel art; `optimizacion-vfx` = presupuesto, overdraw y device real. Regla transversal de todo el material: el cuello del VFX es el **fillrate/overdraw** de transparencias, no el número de partículas ni de nodos. Todo verificado contra Unity 6.x + URP / Shader Graph 17.x / VFX Graph 17.x (estado julio 2026).
