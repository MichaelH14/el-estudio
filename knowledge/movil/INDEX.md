# Base de conocimiento Móvil — Índice

> **Qué es esta carpeta:** la capa de **especialización móvil** del plugin El Estudio. No es una base autónoma: PROFUNDIZA el ángulo teléfono (iOS/Android 2026, Unity 6) sobre las bases generales `unity/`, `pipeline/`, `gamedev/`, `ux-ui/`, `modelado/`, `arte-2d/`, `vfx-shaders/` y `texturizado/` — y por diseño **no repite** lo genérico, lo cruza con `[ver: …]`. Cada archivo abre con su trigger ("Cuando cargar este archivo") y cierra con Reglas prácticas + Errores comunes + Fuentes.

Cubre el ciclo completo de un juego móvil real: **diseñar para el teléfono** (`diseno-movil`) → **presupuesto de arte y UI** (`arte-movil`) → **render y rendimiento en el device** (`unity-movil-graficos`, `unity-movil-rendimiento`) → **cablear input y servicios de tienda** (`unity-movil-input-servicios`) → **monetizar** (`monetizacion-movil`) → **entender el mercado** (`mercado-movil`) → **publicar** (`release-movil`) → **retener y operar** (`retencion-liveops`). Cargar SOLO el/los archivo(s) del problema en mano, nunca la carpeta entera. Hilo conductor de todos: el **device de gama baja, caliente y a los 40 min** es el veredicto (no tu flagship frío), y en móvil el negocio es **adquisición (LTV>CPI)**, no distribución. `[ver: slug]` cruza dentro de esta carpeta; `[ver: unity/…]`, `[ver: gamedev/…]`, `[ver: pipeline/…]`, etc. van a las bases hermanas, que estos archivos asumen leídas y no reproducen.
